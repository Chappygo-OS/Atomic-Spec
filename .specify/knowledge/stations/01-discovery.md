# Station 01: Discovery & Strategy

## 1. Objective

Produce evidence-backed direction on who the customer is, what job is being solved, and the "wedge" (entry point) to avoid building in a vacuum.

## 2. Inputs

- A project folder structure.
- A rough problem statement.
- An initial hypothesis about the buyer.

## 3. Trigger

At the start of a new project or when defining a major new module/pivot.

## 4. The "How-To"

### Competitor Benchmark

- Analyze 5–15 competitors (direct, adjacent, substitutes).
- Create a "paywall map" (what is gated) and "upgrade trigger map".

### ICP & JTBD

- Define the **Ideal Customer Profile (ICP)** (industry, size, constraints like budget/compliance) and **Anti-ICP** (who to avoid).
- Write 3–5 **Jobs To Be Done** in the format: _When [context], I want to [job], so I can [outcome]_.

### Wedge Definition

- Define a single vertical slice that delivers immediate value.
- Format: "For [ICP], we deliver [measurable outcome] by [unique mechanism], unlike [competitor pattern]".

### Metrics Draft

- **North Star Metric (NSM)**
- **Activation event** (measurable "Aha" moment)
- **Retention definitions** (e.g., Weekly Active Organizations)

## 5. Gate Criteria

- [ ] ≥ 5 direct competitors benchmarked.
- [ ] ≥ 10 distinct user pain statements captured with severity tags.
- [ ] ICP includes explicit constraints and an Anti-ICP definition.
- [ ] Top 1–2 "Anchor jobs" selected.
- [ ] Wedge is defined as a measurable outcome achievable in MVP.
