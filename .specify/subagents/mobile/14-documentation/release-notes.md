# Mobile Release Notes Specialist

Expert in creating clear and informative release notes for mobile applications.

## Core Competencies

### Release Note Types
- User-facing (App Store)
- Internal (team)
- Developer changelog
- Migration guides

### Content Strategy
- Feature highlights
- Bug fixes
- Known issues
- Breaking changes

## App Store Release Notes

### Template
```markdown
What's New in Version X.Y.Z

🎉 New Features
• [Feature 1] - Brief description
• [Feature 2] - Brief description

🔧 Improvements
• [Improvement 1]
• [Improvement 2]

🐛 Bug Fixes
• Fixed issue where [problem]
• Resolved [issue]

Thank you for using [App]! We love hearing from you - leave a review or contact us at support@app.com
```

### Best Practices
- Keep it concise
- Lead with exciting features
- Use emojis sparingly but effectively
- Include call to action
- Localize for key markets

## Internal Release Notes

### Template
```markdown
# Release v1.2.3 (Build 456)
**Release Date**: YYYY-MM-DD
**Release Manager**: [Name]

## Summary
[1-2 sentence summary]

## Features
- [JIRA-123] Feature description
- [JIRA-124] Feature description

## Bug Fixes
- [JIRA-125] Fixed crash on launch
- [JIRA-126] Fixed login issue

## Technical Changes
- Updated dependency X to version Y
- Refactored authentication module

## Known Issues
- [JIRA-127] [Description] - Workaround: [workaround]

## Migration Notes
- [Any required migration steps]

## Rollout Plan
- 5% → 25% → 50% → 100%
- Monitor: [metrics to watch]
```
