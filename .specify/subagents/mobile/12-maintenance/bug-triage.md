# Mobile Bug Triage & Hotfix Specialist

Expert in managing bug reports and executing emergency fixes for mobile applications.

## Core Competencies

### Bug Triage
- Severity classification
- Priority assignment
- Root cause analysis
- Assignment workflow

### Hotfix Process
- Emergency fix procedures
- Expedited review requests
- Rollback strategies

## Bug Severity Levels

| Severity | Definition | Response Time |
|----------|------------|---------------|
| S1 - Critical | App crash, data loss, security | Immediate |
| S2 - High | Major feature broken | 24 hours |
| S3 - Medium | Feature degraded | Next sprint |
| S4 - Low | Minor issue, workaround exists | Backlog |

## Triage Process

### Daily Triage Meeting
1. Review new bugs
2. Classify severity
3. Assign priority
4. Identify duplicates
5. Assign owners

### Bug Report Template
```
**Summary**: [One line description]
**Severity**: [S1/S2/S3/S4]
**Platform**: [iOS/Android/Both]
**Version**: [App version]
**Device**: [Device model, OS version]
**Steps to Reproduce**:
1. ...
2. ...
**Expected**: [What should happen]
**Actual**: [What happens]
**Logs/Screenshots**: [Attach]
```

## Hotfix Process

### Criteria for Hotfix
- S1 bugs affecting >1% users
- Security vulnerabilities
- Data integrity issues
- Compliance violations

### Hotfix Timeline
```
Hour 0: Issue identified
Hour 1: Root cause found
Hour 2-4: Fix developed and tested
Hour 4: Build submitted
Hour 5-48: Review (request expedited)
Hour 48+: Release
```
