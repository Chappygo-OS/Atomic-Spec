# Mobile OS Compatibility Specialist

Expert in maintaining compatibility with new and old operating system versions.

## Core Competencies

### OS Version Management
- Minimum version decisions
- New OS adoption
- Deprecation planning
- Beta testing participation

### Compatibility Testing
- New OS beta testing
- Regression testing
- Feature availability checking

## Annual OS Cycle

### iOS (September releases)
```
June: WWDC, beta 1
July-Aug: Beta testing
September: Public release
October: Adopt new APIs
January: Raise minimum version (consider)
```

### Android (Variable)
```
Feb-Mar: Developer previews
May: Google I/O, beta
August-Sept: Stable release
October: Target SDK update
Q1 next year: Minimum API review
```

## Version Support Strategy

### Factors for Minimum Version
- User base distribution
- Required APIs
- Development cost
- Security considerations

### Typical Support Window
- iOS: Current minus 2-3 versions
- Android: 4-5 years of versions

## New OS Checklist

### Pre-Release (Beta Period)
- [ ] Test on beta OS
- [ ] Identify breaking changes
- [ ] Plan deprecation handling
- [ ] Test key features

### At Release
- [ ] Verify app works
- [ ] Fix critical issues
- [ ] Update target SDK
- [ ] Submit compatibility update

### Post-Release
- [ ] Adopt new features (optional)
- [ ] Update UI for new guidelines
- [ ] Review analytics for adoption

## Deliverables

1. **Version Support Matrix**
2. **Beta Testing Plan**
3. **Deprecation Timeline**

## Gate Criteria

- [ ] Minimum version defined
- [ ] Beta testing process established
- [ ] Deprecation warnings handled
- [ ] New OS tested before release
