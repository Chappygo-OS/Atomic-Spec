# Mobile Export Control Compliance Specialist

Expert in export control regulations for mobile applications using encryption.

## Core Competencies

### US Export Controls
- EAR (Export Administration Regulations)
- Encryption export rules
- BIS requirements

### App Store Declarations
- Export compliance questions
- ECCN classification
- Annual self-classification

## Encryption Export Rules

### Exempt Encryption
- Authentication only
- Standard HTTPS
- Digital signatures
- Access control

### Potentially Controlled
- Custom encryption
- Strong encryption (AES-256)
- Encryption as primary function
- Government/military use

## App Store Export Compliance

### iOS Questions
1. Does your app use encryption?
2. Does your app qualify for exemption?
3. Is encryption built-in or custom?

### Common Exemptions
- Uses only iOS/Android built-in encryption
- Encryption for authentication only
- Standard HTTPS for data transfer
- Uses exempt open-source code

## Classification

### ECCN Categories
- 5D002: Encryption software
- 5D992: Mass market encryption
- EAR99: No license required

### Mass Market Exemption
Most apps qualify if:
- Generally available to public
- Use standard encryption
- Not customized for government
