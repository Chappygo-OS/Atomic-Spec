## Screenshot Requirements

### iOS App Store
| Device | Size | Required |
|--------|------|----------|
| iPhone 6.7" | 1290 x 2796 | Yes |
| iPhone 6.5" | 1284 x 2778 | Fallback |
| iPhone 5.5" | 1242 x 2208 | Optional |

### Google Play
| Type | Size | Required |
|------|------|----------|
| Phone | 1080 x 1920 | Yes |
| 7" Tablet | 1200 x 1920 | If supporting |
| 10" Tablet | 1920 x 1200 | If supporting |
| Feature Graphic | 1024 x 500 | Yes |

## Screenshot Strategy

### Common Patterns
1. **Hero shot**: Main value proposition
2. **Feature walkthrough**: Key features
3. **Social proof**: Reviews, awards
4. **Call to action**: Final CTA screenshot

### Best Practices
- First 2 screenshots are most important
- Show real app UI (required by guidelines)
- Add contextual text overlays
- Use consistent branding
- Test different orders

## Video Specifications

### iOS App Preview
- Duration: 15-30 seconds
- Format: H.264, 30fps
- Audio: Optional (AAC)
- No device frames (Apple adds them)

### Android Promo Video
- YouTube video link
- Duration: 30 sec - 2 min recommended
- Feature graphic as thumbnail
