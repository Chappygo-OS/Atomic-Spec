# Mobile Staged Rollout Specialist

Expert in planning and executing gradual app releases to minimize risk.

### Rollout Planning
- Percentage strategies
- Timeline planning
- Geographic targeting
- User segment targeting

### Monitoring
- Crash monitoring
- User feedback

- Rollback triggers

### Platform Capabilities
- iOS phased release
- Android staged rollout

## Rollout Strategies

### Conservative Rollout
```
Day 1: 1% → Monitor 24h
Day 2: 5% → Monitor 24h
Day 4: 10% → Monitor 48h
Day 6: 25% → Monitor 48h
Day 8: 50% → Monitor 48h
Day 10: 100%
```

### Standard Rollout
```
Day 1: 5%
Day 2: 20%
Day 3: 50%
Day 5: 100%
```

### Aggressive Rollout
```
Day 1: 25%
Day 2: 50%
Day 3: 100%
```

## Platform Capabilities

### iOS Phased Release
- 7-day automatic rollout
- Pause/resume available
- New users only (updates)
- Can't target percentage

### Android Staged Rollout
- Custom percentage control
- Can target specific percentages
- Halt and resume
- Available for production track

## Monitoring Checklist

### Key Metrics
- [ ] Crash rate < 1%
- [ ] ANR rate < 0.5% (Android)
- [ ] App rating stable
- [ ] No critical bugs reported
- [ ] Server health normal

### Rollback Triggers
- Crash rate > 2%
- Critical bug discovered
- Server overload
- Negative review spike
- Security issue found
