## Review Notes Template

```
DEMO ACCOUNT:
Email: demo@example.com
Password: Demo123!

SPECIAL INSTRUCTIONS:
1. [Feature X] requires location permission - please allow when prompted
2. To test [Feature Y], navigate to Settings > Feature Y
3. In-app purchase testing: Use sandbox account

ADDITIONAL CONTEXT:
- This app is for [specific use case]
- [Feature Z] requires backend approval, demo data is pre-loaded
- Push notifications will appear after [action]
```

## Common Issues & Solutions

| Issue | Prevention |
|-------|------------|
| Crashes | Thorough testing, crash-free beta |
| Placeholder content | Remove all lorem ipsum |
| Broken links | Verify all URLs work |
| Login required | Provide demo credentials |
| Unclear purpose | Clear app description |
| Permission abuse | Only request needed permissions |

## Rejection Response

### Process
1. Read rejection carefully
2. Understand specific guideline violated
3. Make required changes
4. Write clear resolution summary
5. Resubmit with explanation

### Appeal Tips
- Be professional and concise
- Reference specific guidelines
- Explain your compliance
- Provide evidence if needed

4. **Rejection Response Template**
