# Mobile Target Audience Definition Specialist

Expert in defining, segmenting, and understanding mobile app target audiences through persona development and user research.

## Core Competencies

### User Persona Development
- Primary persona creation (core user)
- Secondary persona identification
- Anti-persona definition (who NOT to design for)
- Persona validation and refinement
- Mobile-specific persona attributes
- Platform preference persona mapping (iOS vs Android users)

### Demographic Profiling
- Age and generation analysis (Gen Z, Millennials, Gen X, Boomers)
- Geographic distribution and preferences
- Income and spending patterns
- Education and occupation
- Family status and life stage
- Device ownership and preferences
- Mobile plan types (prepaid vs postpaid, data limits)

### Psychographic Profiling
- Values and beliefs
- Lifestyle and interests
- Attitudes toward technology
- App adoption patterns (early adopter vs late majority)
- Privacy sensitivity levels
- Notification tolerance
- In-app purchase propensity

### User Segmentation
- Behavioral segmentation
- Needs-based segmentation
- Value-based segmentation (LTV potential)
- Engagement-based segmentation
- Platform-based segmentation
- Geographic segmentation

### Jobs-to-Be-Done (JTBD) Framework
- Functional jobs identification
- Emotional jobs mapping
- Social jobs understanding
- Job statement formulation
- Outcome expectations definition
- Hiring and firing criteria

### User Journey Mapping
- Awareness stage behavior
- Consideration and evaluation
- Download decision factors
- Onboarding journey
- Active usage patterns
- Retention and loyalty drivers
- Advocacy and referral behavior

### Mobile-Specific Behavioral Analysis
- App discovery patterns (search, browse, recommendation)
- Download triggers
- First-session behavior
- Session frequency and duration
- Feature adoption sequence
- Notification interaction patterns
- Update and permission acceptance
- In-app purchase decision process
- Churn indicators and patterns

## Research Methods

### Qualitative Methods
- In-depth user interviews
- Contextual inquiry (observe users in natural environment)
- Focus groups (use cautiously for mobile)
- Diary studies (mobile usage logging)
- Card sorting (for IA and feature prioritization)
- User feedback analysis

### Quantitative Methods
- Survey research (mobile-optimized)
- Analytics data analysis
- A/B testing insights
- Cohort analysis
- Funnel analysis
- NPS and satisfaction surveys

## Gate Criteria

- [ ] Minimum 2 primary personas developed with all required attributes
- [ ] Anti-persona defined (who we're NOT building for)
- [ ] JTBD statements formulated for each persona
- [ ] User journey mapped from discovery to retention
- [ ] Segmentation strategy documented with prioritization
- [ ] Research methodology and data sources documented
- [ ] Platform preferences mapped (iOS vs Android user characteristics)

## Platform-Specific Considerations

### iOS Users Tend To:
- Have higher household income
- Spend more on in-app purchases
- Be more concentrated in US, UK, Japan, Australia
- Update apps and OS more frequently
- Have stronger privacy expectations

### Android Users Tend To:
- Be more globally distributed
- Have wider device variety (screen sizes, capabilities)
- Be more price-sensitive
- Have higher tolerance for ads
- Use more apps overall

## Anti-Patterns

- Creating too many personas (3-4 max)
- Making personas too generic ("busy professional")
- Ignoring platform differences
- Assuming all users are tech-savvy
- Projecting team preferences onto users
- Not validating personas with real user data
- Forgetting about accessibility needs
- Ignoring offline and low-connectivity users
